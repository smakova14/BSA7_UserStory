# BSA7_UserStory

Проект по разработке и согласованию пользовательских историй (User Stories) для системы онлайн-записи барбершопа (BRB).

## 🛠 Навыки
- User Story Mapping
- Acceptance Criteria
- Test Scenarios
- 3C (Card, Conversation, Confirmation)
- INVEST (Independent, Negotiable, Valuable, Estimable, Small, Testable)

## 📋 Описание проекта
В ходе работы я разработала и согласовала пользовательские истории для трёх ролей: менеджер, клиент и мастер. Истории проверены по критериям 3C и INVEST, уточнены в процессе обсуждения с заказчиком, разработчиком и пользователями.

### Что именно было сделано:
1. **User Story для менеджера (US 1–2)**: Описала две потребности менеджера: управление расписанием мастеров (BB_MNG_001) и мониторинг KPI (BB_MNG_002). Для первой разработала критерии приемки, для второй – тестовый сценарий.
2. **User Story для клиента (US 3–4)**: Описала две потребности зарегистрированного клиента: история посещений и настройка профиля (BB_CUS_003), а также оставление отзыва (BB_CUS_004). Для первой – критерии приемки, для второй – тестовый сценарий.
3. **User Story для мастера (US 5–6)**: Описала две потребности мастера: управление личным расписанием (BB_MSR_001) и просмотр статистики доходов (BB_MSR_002). Для первой – критерии приемки, для второй – тестовый сценарий.
4. **Согласование и ревью (US 1–6)**: Провела обсуждение с участниками, играющими роли заказчика, разработчика и пользователя. Проверила каждую историю по принципам 3C (компактность, обсуждение, подтверждение) и INVEST (независимость, обсуждаемость, ценность, оцениваемость, малый размер, тестируемость).
5. **Внесение изменений**: По итогам обсуждения:
   - Разделила US 1 (менеджер) на две независимые истории: управление расписанием и назначение услуг.
   - Разделила US 2 (менеджер) на две: финансовый отчёт и аналитика отзывов.
   - Разделила US 3 (клиент) на две: история посещений и настройка любимого мастера.
   - Упростила US 6 (мастер): убрала диаграммы, оставила только табличные данные (MVP).
   - Дополнила US 5 (мастер) проверкой конфликта с существующими записями.
